Megatron-LM is a single repository that contains two distinct components with different purposes. Understanding which component to use—and how they relate—is essential before you begin.

## Two components, one repository

<Tabs>
  <Tab title="Megatron Core">
    **Megatron Core** (`megatron/core/`) is a composable library of GPU-optimized building blocks for training large language models. It exposes modular APIs for transformer layers, parallelism primitives, optimizers, datasets, and inference—designed to be embedded inside custom training frameworks.

    Megatron Core is compatible with all NVIDIA Tensor Core GPUs and popular LLM architectures including GPT, BERT, T5, LLaMA, Qwen, Mixtral, and Mamba.

    ```bash
    # Install Megatron Core as a standalone library
    uv pip install megatron-core
    ```

## Who should use which component

| Audience | Recommended entry point | Why |
|---|---|---|
| **Framework developers** embedding Megatron into a larger system (NeMo, custom trainer) | Megatron Core | Composable APIs, no training-script dependencies |
| **ML engineers** building production pipelines | Megatron Core | Modular components, fault tolerance, FSDP support |
| **Research teams** exploring new architectures or training techniques | Megatron-LM | Pre-wired scripts, fast iteration, full examples |
| **Learning distributed training** | Megatron-LM | Best-practice configs, documented reference patterns |
| **Quick experimentation** on known architectures | Megatron-LM | Zero boilerplate, proven configurations |

```
│   ├── gpt3/
│   ├── llama/
│   ├── mixtral/
│   └── post_training/
├── tools/                       # Data preprocessing and conversion utilities
├── tests/                       # Unit and integration test suite
└── docs/                        # Documentation source
```

<Note>
All Megatron Core source lives under `megatron/core/`. If you are writing code that imports from `megatron.core`, you are using Megatron Core. The `megatron/training/` directory and `pretrain_*.py` scripts at the root are part of Megatron-LM.
</Note>

## Megatron Core components in detail

### Transformer building blocks (`megatron/core/transformer/`)

The transformer module provides composable attention, MLP, and normalization layers that can be assembled into full model architectures. Each component is GPU-optimized and integrates with Transformer Engine for FP8 compute.

```python
from megatron.core.transformer import TransformerConfig, TransformerLayer
from megatron.core.models.gpt import GPTModel

config = TransformerConfig(
    num_layers=32,
    hidden_size=4096,
    num_attention_heads=32,
    use_cpu_initialization=False,
    bf16=True,
)
model = GPTModel(config, vocab_size=131072, max_sequence_length=8192)
```

### Parallelism (`megatron/core/tensor_parallel/`, `megatron/core/pipeline_parallel/`)

Parallelism primitives are encapsulated in `parallel_state`, which initializes process groups for TP, PP, DP, CP, and EP. A single `ModelParallelConfig` dataclass controls all parallelism degrees and their interactions.

```python
from megatron.core import parallel_state
from megatron.core.model_parallel_config import ModelParallelConfig

parallel_state.initialize_model_parallel(
    tensor_model_parallel_size=4,
    pipeline_model_parallel_size=4,
    context_parallel_size=2,
)
```

### Distributed optimizer (`megatron/core/distributed/`)

The distributed optimizer shards optimizer states across data-parallel ranks, significantly reducing per-GPU memory. Megatron FSDP extends this to also shard gradients and parameters (ZeRO-1/2/3).

### Datasets (`megatron/core/datasets/`)

Megatron Core includes a high-performance indexed dataset format and blending utilities for mixing multiple data sources at configurable sampling ratios. For multimodal workloads, the Megatron Energon library provides text, image, video, and audio loading.

### Inference (`megatron/core/inference/`)

An inference engine and HTTP server for serving trained models, with support for dynamic batching and KV-cache management. Models can also be exported to TensorRT-LLM for optimized deployment.

## Ecosystem

Megatron Core sits at the center of a larger ecosystem:

| Library | Role |
|---|---|
| **[Transformer Engine](https://github.com/NVIDIA/TransformerEngine)** | FP8 kernels and optimized attention used by Megatron Core internally |
| **[Megatron Energon](https://github.com/NVIDIA/Megatron-Energon)** | Multimodal data loader (text, images, video, audio) |
| **[NVRx](https://github.com/NVIDIA/nvidia-resiliency-ext)** | Fault-tolerant training with failure detection and automatic recovery |
| **[Megatron Bridge](https://github.com/NVIDIA-NeMo/Megatron-Bridge)** | Bidirectional Hugging Face ↔ Megatron checkpoint conversion |
| **[NeMo Framework](https://docs.nvidia.com/nemo-framework/user-guide/latest/overview.html)** | Enterprise training framework built on Megatron Core |
| **[NeMo RL](https://github.com/NVIDIA-NeMo/RL)** | Post-training with RLHF, DPO, and other alignment methods |
| **[ModelOpt](https://github.com/NVIDIA/Model-Optimizer)** | Quantization, pruning, distillation, and speculative decoding |
