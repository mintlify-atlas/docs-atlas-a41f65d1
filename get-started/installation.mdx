## System Requirements

### Hardware

| Component | Requirement |
|---|---|
| GPU architecture | NVIDIA Turing or later (recommended) |
| FP8 / FP4 training | NVIDIA Hopper, Ada, or Blackwell GPUs required |

### Software

| Component | Requirement |
|---|---|
| Python | >= 3.10 (3.12 recommended) |
| PyTorch | >= 2.6.0 |
| CUDA Toolkit | Latest stable version |

<Warning>
  Python 3.10 support is being dropped in the upcoming 0.17.0 release. New projects should target Python 3.12 to stay compatible with future Megatron Core releases.
</Warning>

## Option A: PyPI Install (Recommended)

<Steps>
  <Step title="Install uv">
    [uv](https://docs.astral.sh/uv/) is a fast Python package installer and the recommended way to manage your Megatron environment.

    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
  </Step>

  <Step title="Install Megatron Core">
    Install the latest stable release from PyPI:

    <CodeGroup>
      ```bash Core only
      uv pip install megatron-core
      ```

      ```bash With training extras
      uv pip install "megatron-core[training]"
      ```

      ```bash With all dev extras
      uv pip install --group build
      uv pip install --no-build-isolation "megatron-core[training,dev]"
      ```
    </CodeGroup>

    The `[training]` extra adds Weights & Biases, SentencePiece, and Hugging Face Transformers.

    The `[dev]` extra additionally includes [Transformer Engine](https://github.com/NVIDIA/TransformerEngine), `mamba-ssm`, and `nv-grouped-gemm`. These packages compile CUDA kernels at install time, so `torch` must be available in your environment before running this step.

    <Note>
      `--no-build-isolation` is required for `[dev]` because several packages (`mamba-ssm`, `nv-grouped-gemm`, `transformer-engine`) import PyTorch at build time to compile CUDA kernels. Expect this step to take **20+ minutes** depending on your hardware. If you prefer pre-built binaries, the NGC container ships with these pre-compiled.
    </Note>

    <Tip>
      For a lighter dev setup without Transformer Engine and ModelOpt, use `[lts]` instead of `[dev]`:

      ```bash
      uv pip install --no-build-isolation "megatron-core[training,lts]"
      ```

      The `[lts]` and `[dev]` extras are mutually exclusive.
    </Tip>

    <Warning>
      Building from source can consume a large amount of memory. By default the build runs one compiler job per CPU core, which can cause out-of-memory failures on machines with many cores. Limit parallel compilation by setting `MAX_JOBS` before installing:

      ```bash
      MAX_JOBS=4 uv pip install --no-build-isolation "megatron-core[training,dev]"
      ```
    </Warning>
  </Step>

  <Step title="Clone the repository (optional)">
    The PyPI package contains Megatron Core. To access the example training scripts and utilities from Megatron-LM, clone the repository separately:

    ```bash
    git clone https://github.com/NVIDIA/Megatron-LM.git
    ```
  </Step>
</Steps>

## Option B: Install from Source

Use this option for development or to run the latest unreleased code.

<Steps>
  <Step title="Install uv">
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
  </Step>

  <Step title="Clone and install">
    ```bash
    git clone https://github.com/NVIDIA/Megatron-LM.git
    cd Megatron-LM
    uv pip install -e .
    ```

    To include all development dependencies:

    ```bash
    uv pip install --group build
    uv pip install --no-build-isolation -e ".[training,dev]"
    ```

    <Tip>
      If the build runs out of memory, limit parallel compilation jobs:

      ```bash
      MAX_JOBS=4 uv pip install --no-build-isolation -e ".[training,dev]"
      ```
    </Tip>
  </Step>
</Steps>

## Option C: NGC Container

The [PyTorch NGC Container](https://catalog.ngc.nvidia.com/orgs/nvidia/containers/pytorch) provides a pre-configured environment with PyTorch, CUDA, cuDNN, NCCL, and Transformer Engine already compiled and installed. This is the fastest path to running FP8 training without building anything from source.

<Note>
  Use the **previous month's** NGC container rather than the latest one to ensure compatibility with the current Megatron Core release and testing matrix.
</Note>

<Steps>
  <Step title="Pull and start the container">
    ```bash
    docker run --gpus all -it --rm \
      -v /path/to/dataset:/workspace/dataset \
      -v /path/to/checkpoints:/workspace/checkpoints \
      -e PIP_CONSTRAINT= \
      nvcr.io/nvidia/pytorch:26.01-py3
    ```

    <Note>
      The NGC PyTorch container constrains the Python environment using `PIP_CONSTRAINT`. The `-e PIP_CONSTRAINT=` flag unsets this constraint so that Megatron Core and its dependencies install correctly inside the container.
    </Note>
  </Step>

  <Step title="Install Megatron Core inside the container">
    PyTorch is already available in the NGC image, so you can install the full `[dev]` extras without pre-installing build dependencies:

    ```bash
    pip install uv
    uv pip install --no-build-isolation "megatron-core[training,dev]"
    ```
  </Step>

  <Step title="Clone the repository for examples">
    ```bash
    git clone https://github.com/NVIDIA/Megatron-LM.git
    cd Megatron-LM
    ```
  </Step>
</Steps>
